-왜 필요한가
1.지속적인 유지보수에 필요한 백업
2.규모가 큰 프로젝트의 협업에 필요한 공유

-기능
commit - 지금까지 한 작업물(폴더의 전체내용) 백업(박제)
	변화 있을때마다 commit하면 박제
	각 버전의 변경사항들만 박제(용량차지x)

logs - 박제 내역확인

reset --hard 로그 - 박제됐던 상태로 복원
(수정된내용 복원,새로만든파일 삭제, 지웠던 파일 복구)

branch "(브랜치명)" - 시도해볼부분을 작업하는 곳

merge "(브랜치명)" - 브랜치를 메인브랜치에 합침


git - 버전관리를 위한 소프트웨어 (영상을 찍는 앱)
github - git으로 저장되어원격전송된 파일들이 
	저장되는 공간을 제공하는 서비스 (유툽)
